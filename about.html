About Playloop

Playloop is an independent studio building small, fast, fair browser games. Loop Runner is our flagship: a reflex-first arcade title that loads instantly, runs in any modern browser, and treats your time and your hardware with respect.

What "Playloop" means

The name is a deliberate pun on three things. Play loop, the unit of gameplay — the short cycle of input, response, and consequence that arcade games are built from. Loop as in the eternal return: every run is the same shape, every Daily Run resets at midnight, every theme rotation closes back on itself. And loop the dance term, the small repeated motion that becomes a rhythm. Loop Runner's whole design is built around that idea — a few seconds of movement and reaction, repeated until it becomes muscle memory, then sharpened by harder waves.

We chose the .run TLD to lean into it. Playloop dot run — every part of the address points the same direction.

Why Loop Runner exists

The honest version: there are very few games that load instantly on a phone, work without an account, run cleanly on a 7-year-old laptop, and still feel sharp enough to chase a leaderboard. The web used to be full of them — Miniclip in the late 2000s, Kongregate in the early 2010s — and most of that ecosystem evaporated when Flash died. The HTML5 replacements are largely portal-hosted reskins of the same five auto-runner templates.

We wanted to make one that wasn't a template. The brief was: it has to be playable in three seconds, it has to be skill-rewarding not luck-rewarding, it has to have a real leaderboard with real people on it, and it has to be free in the unqualified sense — no premium upgrades, no "watch ad to revive," no paid currency, no account walls. Loop Runner is what came out of that brief.

Everything else followed. The Daily Run is there because a shared seed is the only fair way to compare scores. The eight rotating themes are there because variety keeps long runs interesting without changing the mechanics. The auto-pick upgrade system replaced a card-pick modal because the modal was forcing players to look away from the action at the worst possible moments. Every feature was a response to something specific that wasn't working.

Design principles we work to

1. Respect time

The game loads in under a second on a fast connection and under three on a slow one. There is no splash screen you have to dismiss, no account creation, no tutorial. The Play button is the first thing visible. If we ever add a feature that delays the first interaction by more than half a second, we'll have failed this principle.

2. Respect hardware

Loop Runner runs on Canvas2D, not WebGL. The full asset payload is a few KB of vector descriptions — no sprite atlases, no textures, no audio files larger than a sound effect. On a five-year-old phone it holds 60 FPS in every theme. The May 2026 perf pass specifically rewrote two themes to drop expensive per-frame allocations, because we noticed a few entry-level Androids dropping frames on Cosmic.

3. Respect attention

Ads on the site are placed at the page boundaries (top banner, between-runs interstitial, leaderboard modal). They never overlay gameplay. They never appear mid-run. They never use sound or motion to demand attention. We use Google's Funding Choices CMP so EU/UK users can decline personalised advertising in one click — and the game is identical either way.

4. Respect fairness

There is no way to buy advantage. Every player gets the same upgrade pool. The Daily Run uses a shared seed so identical attempts are directly comparable. The leaderboard is a single worldwide pool — no separate paid-tier rankings, no boost-active filter, no anything that distinguishes a player's money from their skill.

5. Respect intelligence

We don't gate explanations behind tutorials. The Strategy Guide exists for players who want to dig into combo theory; the Themes reference documents every enemy in detail. Nothing in the game is hidden mechanics — the HUD shows every relevant number, and the on-screen feedback (debris colour, decay-timer flicker, boss HP bar) communicates everything else.

How Loop Runner is made

The whole game is plain HTML, CSS, and vanilla JavaScript — no frameworks, no build step, no bundler. The full source lives in a single game.js file. The art is all custom vector code: every enemy in every theme is drawn live from a function, not loaded as an image. That's why the file is small, that's why a theme rotation is cheap, and that's why we can redraw 48 distinct creatures without ever shipping a sprite atlas.

Hosting is GitHub Pages. The leaderboard backend is Supabase — a Postgres table behind a REST API, with row-level security for write rules. Geolocation flags use a free public IP service. No tracking pixels beyond the AdSense and a privacy-friendly analytics counter (Plausible) that counts pageviews without setting cookies or fingerprinting visitors.

The site is built and maintained by one person, working evenings. Updates ship when they're ready; there is no marketing calendar. The devlog covers every release in detail.

What's on the roadmap

No promises and no dates, but the next things on the list:

  • More themes. Eight is the rotation we ship with, but there's no architectural reason it has to stop there. New themes plug in as a single function and a colour palette. The hard part isn't the code, it's making each new theme feel genuinely distinct rather than a reskin.
  • Better mobile. The current touch controls work but feel imprecise compared to mouse. We want to ship a control mode that's specifically tuned for thumbs rather than just adapted from desktop.
  • Portal distribution. Loop Runner is on the way to CrazyGames, GameDistribution, Y8, and other browser-game portals. The portal versions are mechanically identical to the playloop.run version and share a single worldwide leaderboard.
  • Accessibility pass. Reduced-motion mode, keyboard-only input alternative for players who can't use a pointing device, larger-text option for the HUD. These are real gaps in the current build.
  • Replay sharing. A shareable seed format so you can post "here's exactly the run I just had" and let someone else play the same spawn order.

None of these are committed. The actual priority order is whatever feels worst on the current build at the time we sit down to work on it.

What we won't do

Worth being explicit about, because the browser-game space is full of patterns that fail Loop Runner's principles:

  • No premium currency or paid upgrades. Not now, not later. The leaderboard is honest because nobody can buy advantage.
  • No "watch an ad to revive." Death is final in the run that just ended. Tapping an ad for a second chance corrupts the score, the leaderboard, and the loop.
  • No account creation. A display name on the start overlay is the only identity the game needs.
  • No notification permissions. The site will never ask to send you push notifications.
  • No autoplay video. The ad placements are static. No mid-game video roll, no autoplay sound, no anything that breaks immersion.
  • No fake "limited time" pressure. The Daily Run is a real shared seed, not a marketing event. Nothing on the site uses countdown timers to manufacture urgency.
